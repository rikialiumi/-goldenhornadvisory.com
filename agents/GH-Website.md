## Purpose

## Site Architecture

### Files
```
golden-horn/website/
├── index.html          # Main single-page site (all sections)
├── reports.html        # Reports/downloads page
├── privacy-policy.html # Privacy policy
├── gha-logo.png        # Logo asset
└── CNAME               # goldenhornadvisory.com
```

### Key Technical Facts
- **Single HTML file** — all CSS, HTML, and JS are inline in `index.html`
- **GitHub Pages** — changes go live when pushed to `main` branch
- **Bilingual** — every user-facing string must exist in BOTH the `en` and `tr` sections of the `T` object in the inline JavaScript
- **No build system** — edit HTML directly, no npm, no bundler, no framework
- **Formspree** — contact form endpoint: `https://formspree.io/f/xzdkwbzg`
- **No Cloudflare email obfuscation** — email is generated via JS: `info@goldenhornadvisory.com`

### CSS Architecture
- All CSS is in the `<style>` block in `<head>`
- CSS variables defined in `:root` — always use these, never hardcode colours
- Key variables: `--navy:#0A1628`, `--gold:#C4984A`, `--gold-light:#D4AD6A`, `--cream:#F7F2E8`, `--white:#FDFAF5`

### JavaScript Architecture
- All JS is in a single `<script>` block at the bottom of `<body>`
- `T` object contains all translatable strings: `T.en` and `T.tr`
- `setLang(l)` — switches all text to `en` or `tr`
- `toggleFaq(el)` — FAQ accordion
- `handleSubmit(e)` — contact form submission to Formspree
- `acceptCookie()` / `declineCookie()` — cookie banner
- `toggleMenu()` — mobile nav
- IntersectionObserver — scroll reveal for cards
- Parallax + nav shrink on scroll

### Section IDs (in order)
`#hero` → `#services` → `#process` → `#sectors` → `#about` → `#blog` → `#pricing` → `#faq` → `#contact` → `<footer>`

## Core Rules — Never Break These

1. **Always maintain bilingual support.** Any new text element must have an `id="k-*"` attribute and a corresponding entry in both `T.en` and `T.tr`
2. **Never break the contact form.** The Formspree fetch in `handleSubmit()` must remain intact
3. **Never hardcode the email address in HTML** — it is generated by the JS IIFE at the bottom of the script block
4. **Never remove the cookie banner** — it is required for GDPR compliance
5. **Always commit and push to GitHub** after any change — the site goes live only on push to `main`
6. **Always read the file before editing** — never make blind edits
7. **Test logic** — after JS changes, mentally trace the function to verify no syntax errors
8. **CSS variables only** — never hardcode `#0A1628` or `#C4984A` directly; use `var(--navy)` and `var(--gold)`

## Translation Key Naming Convention

All translation keys follow this pattern: `k-[section]-[element]`

**Examples:**
- `k-hero-title` — hero section title
- `k-svc-label` — services section label
- `k-cnt-intro` — contact section intro

When adding a new element:
1. Add `id="k-[section]-[element]"` to the HTML element
2. Add `'k-[section]-[element]': 'English text'` to `T.en`
3. Add `'k-[section]-[element]': 'Türkçe metin'` to `T.tr`

## Workflow

### For a content update:
1. Read `golden-horn/website/index.html` (or relevant file)
2. Identify the element to change
3. Make the HTML change
4. Update `T.en` and `T.tr` in the JS T object if the text is translatable
5. Verify no syntax errors in the change
6. Commit: `content: [description of change]`
7. Push to `main`

### For a bug fix:
1. Read the file and identify the root cause
2. Fix the minimal necessary code — do not refactor surrounding code
3. Verify the fix doesn't break adjacent functionality
4. Commit: `fix: [description of bug and fix]`
5. Push to `main`

### For a new page:
1. Create the new HTML file in `golden-horn/website/`
2. Match the design: same fonts (Cormorant Garant + DM Sans), same colour variables, same nav + footer pattern
3. Inline all CSS — no external stylesheets
4. Add the nav link in `index.html` navigation and update `T.en`/`T.tr`
5. Commit both files together
6. Push to `main`

### Git commit message format:
```
[type]: [short description]

[optional longer explanation]

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
```

Types: `fix`, `content`, `style`, `feat`, `refactor`
